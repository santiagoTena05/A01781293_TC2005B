  2. ¿Qué es un nodo?

    3. ChatGPT:

    En el contexto de JavaScript y el modelo de objeto de documento (DOM), un nodo es un objeto que representa un elemento individual en la estructura de un documento HTML o XML. Existen diferentes tipos de nodos en el DOM, como nodos de elemento (representan las etiquetas HTML/XML), nodos de texto (representan el contenido de texto dentro de un elemento) y nodos de atributo (representan los atributos de un elemento). Los nodos en el DOM se organizan en una estructura de árbol, donde cada nodo puede tener uno o varios nodos secundarios (hijos) y un nodo padre. Esta estructura de árbol se utiliza para representar la estructura jerárquica de un documento HTML o XML. El acceso y manipulación de nodos en el DOM es una parte fundamental de la programación web con JavaScript, ya que permite la creación dinámica de contenido y la interacción con el usuario a través de eventos y acciones del usuario.

    Copilot:

    // r: Un nodo es un elemento del árbol DOM.

  5. ¿Cuál es la diferencia entre getElementby* y querySelector*?

    6. ChatGPT:

    La principal diferencia entre los métodos getElementby* y querySelector* es que el primero utiliza la sintaxis de acceso por nombre de etiqueta, clase o ID, mientras que el segundo utiliza selectores CSS para seleccionar elementos del DOM.
    getElementby* incluye los siguientes métodos:
    getElementById(): devuelve el elemento con el ID especificado. getElementsByClassName(): devuelve una lista de elementos con la clase especificada. getElementsByTagName(): devuelve una lista de elementos con el nombre de etiqueta especificado.

  6. ¿Qué es un evento?

    7. ChatGPT:

    En JavaScript, un evento es cualquier acción que ocurre en una página web, como hacer clic en un botón, mover el mouse, presionar una tecla, cargar una página, entre otros. Los eventos son fundamentales para la programación web, ya que permiten que el usuario interactúe con la página y que la página responda a esas interacciones.

    Copilot:

    // r: Es una acción que se produce en el navegador, como por ejemplo el click de un botón.

  7. ¿Qué es un listener?

    8. ChatGPT:

    En JavaScript, un listener o "escuchador" es una función que se encarga de esperar y responder a un evento específico. Un listener está vinculado a un elemento HTML y se activa cuando ocurre un evento en ese elemento. Para agregar un listener a un elemento HTML, se utiliza el método addEventListener que acepta tres argumentos: el tipo de evento que se desea escuchar (como 'click', 'mouseover', 'submit', etc.), la función que se debe ejecutar cuando ocurra el evento y un valor booleano opcional que indica si se debe usar la captura de eventos o la propagación de eventos.

    Copilot:

    // r: Es una función que se ejecuta cuando se produce un evento.

  8. ¿Qué es un callback?

    9. ChatGPT:

    En JavaScript, un callback es una función que se pasa como argumento a otra función y que se ejecuta cuando la función original ha terminado de procesar su trabajo. El callback se utiliza para notificar que se ha completado una tarea y para proporcionar el resultado de esa tarea.

    Copilot:

    // r: Es una función que se pasa como parámetro a otra función.

  9. ¿Qué es un objeto de evento?

    10. ChatGPT:

    En JavaScript, un objeto de evento es un objeto que contiene información sobre un evento que acaba de ocurrir en la página web. Cuando un evento ocurre en un elemento HTML, como hacer clic en un botón, el navegador crea un objeto de evento que contiene información sobre el evento, como qué tipo de evento fue, qué elemento HTML lo provocó, la posición del mouse, entre otros.
